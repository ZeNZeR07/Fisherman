# โครงสร้างและฟังก์ชันระบบเว็บบอร์ดแสดงผลการแข่งขันตกปลา (Admin-Only Platform)

เอกสารฉบับนี้กำหนดโครงสร้างข้อมูล หน้าเว็บ (UI/UX Flow) และฟังก์ชันการทำงานของระบบเว็บแอปพลิเคชันสำหรับจัดการแข่งขันตกปลา โดยออกแบบมาให้ใช้งานง่าย ระบบไม่ซับซ้อน และโฟกัสที่การทำงานของ Admin เป็นหลัก

---

## 🗺️ ภาพรวมโครงสร้างของเว็บไซต์ (Sitemap Overview)

```
[ Page 1: Login ]
       │
       ▼
[ Page 2: Admin Dashboard ]  ─── (Topbar: + เพิ่มการแข่งขันใหม่)
       │
       ├─► คลิกดูแมตช์ที่ "กำลังแข่งขัน" ──► [ Page 3A: Live Scoring Control Room ]

```

## 📄 รายละเอียดโครงสร้างแต่ละหน้าและฟังก์ชัน (Page-by-Page Specifications)

### Page 1: Login Page (หน้าเข้าสู่ระบบ)
*หน้าแรกสำหรับคัดกรองสิทธิ์ ออกแบบให้เรียบง่ายที่สุดเพื่อความปลอดภัย*

* **โครงสร้างหน้าจอ (Layout):**
    * กล่องฟอร์มล็อกอินอยู่กึ่งกลางหน้าจอ (Centered Card)
    * โลโก้ระบบหรือชื่อเว็บแข่งขันตกปลา
* **ส่วนประกอบบนหน้าจอ (UI Elements):**
    * ช่องกรอกชื่อผู้ใช้ (Username Field)
    * ช่องกรอกรหัสผ่าน (Password Field)
    * ปุ่มเข้าสู่ระบบ (Login Button)
* **ฟังก์ชันการทำงาน (Functions):**
    * **Authentication Check:** ตรวจสอบรหัสผ่านของแอดมิน หากถูกต้องให้ทำการสร้าง Session และพาไปยังหน้า Dashboard (หากไม่ผ่านให้แสดง Error แจ้งเตือน)
    * **Route Guard:** บล็อกไม่ให้ผู้ที่ไม่ได้ล็อกอินพิมพ์ URL เข้าถึงหน้าอื่นๆ ได้โดยตรง

---

### Page 2: Admin Dashboard (หน้าหลักจัดการแข่งขัน)
*ศูนย์กลางการจัดการแมตช์ทั้งหมด แอดมินจะมองเห็นภาพรวมว่ามีทัวร์นาเมนต์ไหนบ้างและสถานะเป็นอย่างไร*

* **โครงสร้างหน้าจอ (Layout):**
    * **Top Navigation Bar:**
        * ด้านซ้าย: โลโก้ และ ชื่อระบบ
        * ด้านขวา: ปุ่ม **`+ เพิ่มการแข่งขันใหม่`** (Highlight Color) และปุ่ม Logout
    * **Main Content Area:** แสดงรายการการแข่งขันแบ่งตามสถานะด้วยการจัดหมวดหมู่ หรือระบบกรองแท็บ (Tabs)
* **ส่วนประกอบบนหน้าจอ (UI Elements):**
    * **Section 1: สถานะ "กำลังดำเนินการ" (Live Matches)** -> การแข่งขันที่กำลังจัดอยู่ ณ ปัจจุบัน
    * **Section 2: สถานะ "สิ้นสุดแล้ว" (Completed Matches)** -> ประวัติการแข่งขันที่จบไปแล้ว
    * **Match Cards (การ์ดรายละเอียดแต่ละแมตช์):** ในการ์ดจะแสดง:
        * ชื่อแมตช์การแข่งขัน / ครั้งที่จัด
        * ชื่อบ่อตกปลาที่เป็นสถานที่จัด
        * วันที่และเวลาที่จัด
        * ป้ายสถานะ (Badge Status): 🔴 กำลังแข่ง (Live) | 🟢 สิ้นสุดแล้ว (Completed)
* **ฟังก์ชันการทำงาน (Functions):**
    * **Popup / Modal - Create New Match:** เมื่อกดปุ่มบน Navbar จะมีหน้าต่างป็อปอัปขึ้นมาให้กรอกข้อมูลเพื่อเปิดแมตช์ใหม่:
        * *ฟิลด์ข้อมูล:* ชื่อแมตช์, บ่อตกปลา, วันที่จัด, และกติกาการคิดคะแนนเบื้องต้น (เช่น น้ำหนักรวม หรือ คะแนนตามชนิดปลา)
    * **Redirection Link:** เมื่อคลิกที่ตัวการ์ดการแข่งขัน ระบบจะอ่านสถานะ (Status) ของแมตช์นั้นอัตโนมัติ:
        * ถ้าแมตช์เป็น `Live` -> พาไปที่ **Page 3A**
        * ถ้าแมตช์เป็น `Completed` -> พาไปที่ **Page 3B**

---

### Page 3A: Live Scoring Control Room (หน้าบันทึกคะแนนระหว่างแข่ง)
*หน้าสำหรับแก้ไขปัญหาจุดลงทะเบียนข้อมูลปลา ออกแบบแอปพลิเคชันเป็นลักษณะ 2 คอลัมน์ทำงานพร้อมกันในหน้าเดียวโดยไม่ต้องเปลี่ยนหน้า*

* **โครงสร้างหน้าจอ (Layout):**
    * **Top Area:** แสดงชื่อบ่อตกปลา, สถานะแมตช์ (Live), และปุ่ม **`🔴 จบการแข่งขัน`** (มุมขวาบน)
    * **Two-Column Layout:** แบ่งหน้าจอเป็นฝั่งซ้าย (แบบฟอร์มคีย์ข้อมูล) และฝั่งขวา (กระดานคะแนนเรียลไทม์)
* **ส่วนประกอบและฟังก์ชันฝั่งซ้าย: ฟอร์มบันทึกน้ำหนักปลา (Data Entry Form)**
    * **ช่องเลือกตัวนักกีฬา:** Dropdown List หรือช่องพิมพ์ค้นหาด่วนด้วย หมายเลขซุ้ม/เบอร์เบ็ด หรือชื่อนักกีฬา
    * **ช่องเลือกชนิดของปลา:** Dropdown หรือปุ่มกดเลือกด่วน (Quick-select) เช่น ปลานิล, ปลาสวาย, ปลาบึก, ปลายี่สก
    * **ช่องกรอกน้ำหนักปลา:** Input Field สำหรับระบุตัวเลขทศนิยม (กิโลกรัม)
    * **ปุ่มบันทึกข้อมูล (Submit Button):** เมื่อแอดมินกดบันทึก:
        * *ระบบจะคำนวณคะแนนให้อัตโนมัติหลังบ้าน* โดยอิงตามน้ำหนักปลา และตัวคูณคะแนนตามประเภทของปลา (ถ้ามี)
        * เพิ่มข้อมูลเข้าสู่ระบบฐานข้อมูลทันทีโดยไม่ต้องรีเฟรชหน้าเว็บ (Real-time Push)
        * เคลียร์ฟอร์มให้ว่างเพื่อรอคีย์ปลาตัวต่อไปได้ทันที
* **ส่วนประกอบและฟังก์ชันฝั่งขวา: กระดานคะแนนสด (Live Leaderboard)**
    * **ตารางคะแนนสะสม (Scoreboard Table):** แสดง อันดับ, หมายเลขซุ้ม/ชื่อ, น้ำหนักปลารวมทั้งหมด, และคะแนนรวม ณ ปัจจุบัน
    * **Auto-Sorting:** ระบบจะทำการจัดเรียงอันดับ (Ranking) ใหม่โดยอัตโนมัติจากคะแนนสูงสุดลงมาต่ำสุดทุกครั้งที่มีการกดบันทึกคะแนนจากฝั่งซ้าย
* **ฟังก์ชันควบคุมแมตช์ (Match Control):**
    * **End Match Button:** เมื่อหมดเวลาแข่งขัน แอดมินกดปุ่ม "จบการแข่งขัน" ระบบจะทำการล็อกหน้าจอฝั่งซ้ายไม่ให้กรอกคะแนนเพิ่มได้อีก และเปลี่ยนสถานะแมตช์ในระบบเป็น `Completed` จากนั้นจะเปลี่ยนการแสดงผลเข้าสู่โหมดสรุปผลชนะเลิศ

### Page 3B: Final Match Leaderboard (หน้าสรุปผลการแข่งขัน)
*หน้านี้มีไว้สำหรับแสดงผลรางวัลอย่างเป็นทางการและดูสถิติย้อนหลัง ไม่สามารถแก้ไขข้อมูลได้แล้ว (Read-Only)*

* **โครงสร้างหน้าจอ (Layout):**
    * **Top Banner (Podium Section):** ส่วนบนสุดแสดงโพเดียมรางวัลอย่างสมเกียรติ สำหรับผู้ชนะเลิศอันดับ 1, 2 และ 3
    * **Bottom Section:** ตารางอันดับผู้เข้าแข่งขันทั้งหมด และข้อมูลเชิงสถิติของแมตช์
* **ส่วนประกอบบนหน้าจอ (UI Elements):**
    * **Podium Highlight Display:** การ์ดหรือบล็อกสีพิเศษ (ทอง, เงิน, ทองแดง) แสดงชื่อผู้ชนะ อันดับ 1-3 พร้อมสรุปคะแนนและน้ำหนักปลาที่ทำได้
    * **Final Ranking Table:** ตารางอันดับตั้งแต่ 4 ลงไปจนถึงอันดับสุดท้าย แสดงประวัติการตกได้ทั้งหมดของทุกคน
    * **Match Statistics (สถิติประจำแมตช์):**
        * ปลาตัวที่น้ำหนักมากที่สุดในเกม (Big Game) และผู้ที่ตกได้
        * จำนวนปลาที่ตกได้รวมทั้งหมดในทัวร์นาเมนต์
        * น้ำหนักปลารวมทั้งหมดของบ่อในวันนั้น
* **ฟังก์ชันการทำงาน (Functions):**
    * **Print / Export Result (Optional):** ปุ่มสำหรับสั่งพิมพ์หน้าสรุปผลนี้ออกเป็นกระดาษหรือเซฟเป็นไฟล์ PDF เพื่อเก็บเป็นรายงานประจำบ่อตกปลา
    * **Back to Dashboard:** ปุ่มย้อนกลับไปหน้าหลักเพื่อเปิดดูแมตช์อื่นๆ

## 📊 ตัวอย่างการจัดเก็บโครงสร้างข้อมูลคะแนนปลา (Score Calculation Logic)

เพื่อให้ระบบคำนวณได้อย่างถูกต้อง ฟังก์ชันหลังบ้านจะคิดคะแนนตามเงื่อนไขนี้:

$$	ext{คะแนนที่ได้ต่อครั้ง} = 	ext{น้ำหนักปลา (กก.)} 	imes 	ext{ค่าน้ำหนักคะแนนของปลาชนิดนั้น (Multiplier)}$$

*ตัวอย่าง:*
* หากตั้งกติกาว่า ปลานิลคิดคะแนนปกติ (Multiplier = 1) -> ตกปลานิลได้ 2.5 กก. = **2.5 คะแนน**
* หากตั้งกติกาว่า ปลาบึกตกยากกว่า (Multiplier = 2) -> ตกปลาบึกได้ 5.0 กก. = **10 คะแนน**

ระบบจะนำคะแนนจากการบันทึกแต่ละครั้งในหน้า **Page 3A** มารวมกันเพื่ออัปเดตกระดานคะแนนแบบเรียลไทม์
